# Credit Risk Model Suite & Excel Integration

End-to-end credit risk modeling platform built on [Domino Data Lab](https://www.dominodatalab.com/). Trains, registers, and serves probability-of-default, expected-loss, and loan-inventory models as REST APIs, then auto-generates an Excel add-in so analysts can call them directly from spreadsheets.

## Architecture

```
expected_loss_models/       Credit risk MLflow pyfunc models
discover_endpoints_and_build_xll.py                  Excel-DNA add-in generator
```

### Credit Risk Models (`expected_loss_models/`)

Four MLflow `pyfunc` models form a composable pipeline:

| Model | File | Description |
|---|---|---|
| **GetCreditCurves** | `credit_curve_model.py` | Builds risk-free + spread curves for a given valuation date |
| **GetLoanProbabilityOfDefault** | `loan_pd_model.py` | XGBoost classifier predicting default probability at a given tenor |
| **GetExpectedLoss** | `expected_loss_model.py` | Computes expected loss, discounted EL, and Basel III risk-weighted assets |
| **GetLoanInventory** | `loan_inventory_model.py` | Generates a synthetic loan portfolio for testing and demonstration |

Supporting files:

| File | Purpose |
|---|---|
| `train_pd_model.py` | Trains the XGBoost PD model and saves `pd_model.json` |
| `synthetic_data.py` | Generates synthetic loan data for training |
| `register_models.py` | Trains the PD model, registers all four models to MLflow, and creates Domino Model API endpoints |
| `test_models.py` | Smoke tests exercising the full pipeline end-to-end |

### Excel UDF Generator (`discover_endpoints_and_build_xll.py`)

Discovers all Domino Model API endpoints for the project, extracts credentials and MLflow parameter signatures, and generates a complete Excel-DNA `.xll` add-in with strongly-typed User Defined Functions for each endpoint.

## Getting Started

### Prerequisites

- Python 3.10+
- A Domino Data Lab workspace (recommended) or the following environment variables set locally:
  - `DOMINO_URL` — Domino deployment URL
  - `DOMINO_USER_API_KEY` — API key for authentication
  - `DOMINO_PROJECT_ID` — Target project ID

### Running in Domino (Recommended)

All commands should be executed as **Domino Jobs** for full governance, reproducibility, and audit trail.

```bash
# Train the PD model
cd expected_loss_models && python train_pd_model.py

# Register all models and create API endpoints
cd expected_loss_models && python register_models.py

# Validate the full pipeline
cd expected_loss_models && python test_models.py

# Generate the Excel add-in
python discover_endpoints_and_build_xll.py
```

### Using the Excel Add-in

1. Run `python discover_endpoints_and_build_xll.py` inside Domino to generate the `.xll` file.
2. Download the generated `DominoModelFunctions-AddIn64.xll` (64-bit) or `DominoModelFunctions-AddIn.xll` (32-bit).
3. Open Excel and load the add-in via **File > Options > Add-Ins > Browse**.
4. Each model endpoint appears as a worksheet function with full parameter documentation.

## Model Details

All models accept column aliases for convenience (e.g., `fico` maps to `credit_score`, `ltv` maps to `loan_to_value_ratio`). Numeric inputs passed to Domino Model APIs are transmitted as strings and coerced internally.

The PD model (`pd_model.json`) is a trained artifact generated by `train_pd_model.py`. It is not checked into version control — run training or registration to regenerate it.
